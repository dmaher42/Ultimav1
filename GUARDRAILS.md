# The Britannia Manifesto: Guardrails for Ultima V1

This document outlines the core principles of development for this remake. Every feature, quest, and map added to the project must be evaluated against these guardrails.

## 1. Moral Complexity Over Binary Choice
*   **Principle**: Actions should have nuanced consequences. The Gargoyles should not be "evil monsters," but a civilization with their own valid grievances.
*   **Guardrail**: Avoid "Kill X to win" quests. Always include a path for empathy, dialogue, or alternative resolution.

## 2. Seamless Exploration
*   **Principle**: The world should feel like a single, contiguous place.
*   **Guardrail**: Favor edge-of-map transitions ("Edge Warping") over disconnected teleportation gates. The world should be traversable from one corner to the other without constant menu-driven travel.

## 3. Depth of Worldbuilding (NPCs & Logic)
*   **Principle**: The world is lived-in. NPCs have names, jobs, and opinions.
*   **Guardrail**: NPCs must use the **Keyword Dialogue System** (Name, Job, Bye, and specific lore keywords). Avoid generic "bark" lines. Quest logic should be driven by information gained from dialogue, not just markers.

## 4. Authentic Britannia
*   **Principle**: This is *Britannia*, not a generic fantasy setting.
*   **Guardrail**: Use canonical locations (Britain, Moonglow, Lycaeum) and lore characters. Avoid anachronistic or generic names (e.g., "Athens"). Content should look, feel, and sound like a 16-bit evolution of the classic Ultima aesthetic.

## 5. Architectural Stability
*   **Principle**: Push content and structure, but respect the existing renderer and runtime constraints.
*   **Guardrail**: Optimize for the current Canvas-based 2D renderer. Ensure all new assets (sprites/tiles) match the established 3x4 frame and variation formats.
